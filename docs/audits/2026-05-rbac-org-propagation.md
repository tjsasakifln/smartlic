## Methodology

1. Walks every `.py` file in `backend/routes/` (recursive).
2. AST-parses each function decorated with `@router.<method>(...)`.
3. Classifies on three axes:
   - `accepts_org_id`: `{org_id}` in path OR `org_id` in parameter list
   - `touches_org_table`: references `.table("organizations")` or `.table("organization_members")`
   - `enforces_role`: any default-arg uses `Depends(require_org_role(...))`
4. Severity matrix:
   - P0 = accepts org_id + touches table + NO require_org_role
   - P1 = accepts org_id + NO require_org_role (regardless of table)
   - P2 = touches table + does not accept org_id (no obvious leak, review)
   - OK = exempt module OR fully enforced OR no org context

## Allow-list
