# UX-420 — Falta de Acentuação em Labels do App

**Status:** Done
**Severity:** MEDIUM
**Origin:** Auditoria UX Playwright 25/03/2026
**Agent:** @ux-design-expert (Uma)

## Problema

Múltiplos labels sem acentuação correta:
- Menu Conta: "Seguranca" → Segurança
- Footer busca: "Politica de Privacidade" → Política, "Precos" → Preços
- Footer: "Avaliacoes" → Avaliações

## Impacto

Transmite falta de polimento para produto pago (R$297/mês).

## Nota

Story UX-353 já abordou acentuação parcialmente. Esta story cobre resíduos encontrados na auditoria.

## Acceptance Criteria

- [x] AC1: Todos os labels visíveis no app usam acentuação correta do português
- [x] AC2: Busca textual por padrões comuns: "Seguranca", "Politica", "Precos", "Avaliacoes", "Historico" sem acentos
- [x] AC3: Footer interno e externo com acentuação correta
- [x] AC4: Menu de conta com acentuação correta
