## Lições aprendidas

### Padrões empíricos validados

1. **Sweep multi-arquivo single PR Sentry-priorized > fix-per-route** (Stage 4+5 confirmaram 2x)
2. **WC bump separado de sweep + 24h soak post-merge** (memory `feedback_sweep_single_pr_required`)
3. **Discriminator empírico barato (<5min) ANTES de ação** (memory `feedback_advisor_critical_discernment`)
4. **Schema drift requer query dual source-of-truth ANTES de speculative investigation** (memory `feedback_schema_drift_psql_discriminator`)
5. **Warm continuation Stage 5+ não pivota — band-aid 7× falha; cold session dedicated mandatório** (memory `feedback_chief_warm_stage5plus_no_pivot`)
6. **CRIT-050 silent broken durante drift** — discriminator: rolconfig query Management API antes de NOTIFY pgrst
7. **Caller `asyncio.wait_for` não libera pool até server statement_timeout** — floor 15s não-negociável (8s piorou)

### Anti-patterns reforçados (já em architecture.md §8)

- Sync `.execute()` em route async — bloqueia event loop
- `service_role` sem statement_timeout — pool exhaustion
- Build SSG hammers backend — Promise.all de muitos fetches
- `revalidate=N` + `cache:'no-store'` — quebra SSG (SEN-FE-001 recidiva)

### Capacity reality

- Hobby Plan: **48 vCPU / 48 GB RAM por serviço** (não 1 GB) — memory `reference_railway_hobby_plan_actual`
- WC=4 outcome IDÊNTICO a WC=1 sob bot/cron load — capacity NÃO é constraint
- Constraint é **pattern arquitetural** (sync execute + sem budget + sem negative cache)
